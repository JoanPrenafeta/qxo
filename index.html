Tips

  • No usar Aria és millor que utilitzar-les malament.
  • Utilitza sempre elements HTML semàntics (button, a, input, nav, header, main...) en lloc de recórrer a ARIA.
  • Mai no utilitzis ARIA per canviar el rol inherent d'un element natiu (Ex: No donis role="button" a un <a> amb href).
  • Assegura't que el component tingui una etiqueta visual (amb <label> o <legend>) abans de complementar-la amb ARIA.
  • ARIA només s'ha d'utilitzar per millorar la semàntica d'elements d'interfície d'usuari, no per a elements purament decoratius.
  • Si utilitzes ARIA, valida el codi amb eines com l'Extensió WAVE o l'Inspector d'Accessibilitat del Navegador.
  • Si un element fa una acció (envia, tanca, obre un modal, etc.), ha de ser un <button>.
  • Si un element navega a un URL o baixa un recurs, ha de ser un <a> amb href.
  • Assegura't que tots els elements interactius es puguin enfocar utilitzant només el tabulador (Tab).
  • L'ordre de tabulació (Tab) ha de seguir l'ordre visual i lògic de la pàgina. No enganyis l'usuari!
  • Mai no utilitzis tabindex="1" o superior; això trenca l'ordre natural i confon els usuaris de teclat.
  • Tots els camps de formulari han de tenir un <label> correctament associat amb for i id.
  • Utilitza sempre <fieldset> i <legend> per agrupar i etiquetar un conjunt de botons d'opció o caselles de selecció.
  • El text del placeholder no substitueix mai el <label>. El lector de pantalla no hi confia.
  • Evita text d'enllaç genèric com "Fes clic aquí" o "Més". Fes que el text de l'enllaç sigui el destí.
  • Quan utilitzis target="_blank", notifica a l'usuari amb aria-describedby (o text visible) que l'enllaç s'obrirà en una nova pestanya.
  • Utilitza aria-label només si l'element no té text visible (Ex: Icones o botons amb només una icona).
  • Utilitza aria-hidden="true" en elements purament decoratius (icones, imatges de fons, separadors) perquè el lector de pantalla els ignori.
  • Si mostres elements en llista, utilitza les etiquetes semàntiques <ul> o <ol>. No converteixis <div> en llistes.
  • Defineix les seccions principals de la pàgina amb regions landmark d'ARIA (<main>, <nav>, <header>). Això permet la navegació ràpida.
  • Per a contingut que canvia dinàmicament sense recarregar la pàgina (Ex: errors, actualitzacions d'estat), utilitza regions live amb aria-live="polite".
  • Evita el role="application" tret que estiguis creant una aplicació web complexa on la navegació de teclat nativa s'ha de deshabilitar.
  • Quan un element canvia d'estat (Ex: llista desplegable s'obre), actualitza l'estat ARIA corresponent (Ex: aria-expanded="true/false").
  • Assegura't que l'anell de focus que envolta l'element actiu no estigui mai desactivat (outline: none;) amb CSS.
  • Tots els texts han de complir amb una ràtio de contrast respecte al fons d'almenys 4.5:1 (WCAG AA).
  • Evita text incrustat dins les imatges. Si és imprescindible, el text ha de ser inclòs dins el text alternatiu (alt).
  • El disseny ha de permetre que el text es redimensioni almenys fins al 200% sense perdre contingut ni superposar-se.
  • Evita justificar el text amb CSS (text-align: justify), ja que pot crear "rius" d'espai en blanc que dificulten la lectura.
  • Totes les imatges informatives (<img>) han de tenir un atribut alt descriptiu. Les decoratives han de tenir alt="".
  • Defineix l'idioma principal a l'etiqueta <html> (Ex: <html lang="ca">) i els canvis d'idioma locals amb l'atribut lang.
  • Quan hi hagi un error al formulari, informa l'usuari sobre què ha fallat, com solucionar-ho i utilitza aria-describedby amb l'error.
